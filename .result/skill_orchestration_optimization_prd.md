# 技能编排优化产品需求文档 (PRD)

## 项目概述

本文档描述了对百宝箱企业版技能编排功能中意图识别模块的优化需求，主要优化点包括支持添加服务流，以及完善插件添加的提示机制。

## 需求背景

当前技能编排功能中的意图识别模块只支持添加插件，但随着平台服务能力的扩展，用户需要在技能编排中同时支持插件和服务流两种服务类型，以实现更丰富的应用构建能力。

## 当前功能分析

### 现有意图识别模块功能
- **核心作用**：智能体的核心能力模块，大模型会根据用户输入自动判断用户意图并调用对应技能
- **技能配置流程**：
  1. 填写技能名称、调用场景描述、标签
  2. 点击"添加插件"选择功能插件
  3. 配置插件输入/输出参数
  4. 保存意图技能
- **支持多技能**：可在同一意图识别模块下配置多个技能

## 优化需求

### 1. 核心功能扩展

#### 1.1 服务类型支持扩展
- **原功能**：仅支持添加插件
- **新功能**：支持添加插件和服务流两种服务类型

#### 1.2 添加方式优化
- **插件添加**：保持原有添加插件的功能，增加标准协议插件提示
- **服务流添加**：新增服务流选择功能

### 2. 具体功能需求

#### 2.1 服务类型选择界面

**UI设计要求**：
- 在原有"添加插件"按钮基础上，改为两个选项：
  - "添加插件" 按钮
  - "添加服务流" 按钮
- 两个按钮采用并列布局，保持视觉一致性

#### 2.2 插件添加功能优化

**功能保持**：
- 保留原有插件选择框的所有功能
- 保留插件参数配置功能

**新增提示机制**：
- 在插件选择框顶部添加提示信息
- **提示文案**："该功能下，只支持添加符合标准协议的插件"
- **提示样式**：使用信息提示样式，不阻塞用户操作

#### 2.3 服务流添加功能

**选择界面设计**：
- 参考现有插件选择框的设计风格
- 标题显示"选择服务流"
- 显示当前用户空间下的所有可用服务流

**服务流展示内容**：
- 服务流名称
- 服务流简介/描述
- 服务流类型标识
- 创建时间或最后修改时间

**筛选和搜索**：
- 支持按名称搜索服务流
- 支持按类型筛选（如有多种服务流类型）

**选择确认**：
- 单选模式，一次只能选择一个服务流
- 选择后显示确认和取消按钮
- 确认后返回到技能配置界面

#### 2.4 服务流参数配置

**配置界面**：
- 类似插件参数配置界面
- 根据服务流的输入/输出规范进行参数配置

**参数类型**：
- 输入参数配置
- 输出参数映射
- 执行条件设置（如需要）

### 3. 交互流程

#### 3.1 插件添加流程（优化后）
1. 用户点击"添加插件"按钮
2. 弹出插件选择框，顶部显示标准协议提示
3. 用户从符合标准协议的插件中选择
4. 配置插件参数
5. 保存技能配置

#### 3.2 服务流添加流程（新增）
1. 用户点击"添加服务流"按钮
2. 弹出服务流选择框，显示本空间的服务流
3. 用户筛选或搜索目标服务流
4. 选择服务流并确认
5. 配置服务流参数
6. 保存技能配置

### 4. 技术实现要点

#### 4.1 数据获取
- **插件数据**：继续从现有插件库获取符合标准协议的插件
- **服务流数据**：从用户空间的服务流库获取数据

#### 4.2 权限控制
- **插件权限**：保持现有权限机制
- **服务流权限**：确保只显示用户有权限访问的服务流

#### 4.3 参数配置兼容
- 确保插件和服务流的参数配置界面保持一致的用户体验
- 支持不同类型服务的参数映射机制

### 5. 验收标准

#### 5.1 功能验收
- [ ] 意图识别模块支持添加插件和服务流两种类型
- [ ] 插件添加时显示标准协议提示信息
- [ ] 服务流选择框正确显示用户空间内的服务流
- [ ] 服务流参数配置功能正常工作
- [ ] 配置完成的技能能够正常保存和执行

#### 5.2 体验验收
- [ ] 两种添加方式的界面风格保持一致
- [ ] 操作流程简洁明了，无冗余步骤
- [ ] 提示信息清晰易懂，不干扰用户操作
- [ ] 错误情况下有合适的提示和引导

### 6. 风险评估

#### 6.1 技术风险
- **风险**：服务流参数配置复杂度可能较高
- **缓解**：预先调研服务流的参数规范，设计通用的配置界面

#### 6.2 用户体验风险
- **风险**：新增功能可能增加用户操作复杂度
- **缓解**：保持界面简洁，提供清晰的操作指引

### 7. 上线计划

#### 7.1 开发阶段
1. **阶段一**：UI界面改造，支持两种添加方式选择
2. **阶段二**：服务流选择功能开发
3. **阶段三**：服务流参数配置功能开发
4. **阶段四**：集成测试和用户体验优化

#### 7.2 测试验证
- 单元测试覆盖新增功能点
- 集成测试验证完整操作流程
- 用户体验测试确保操作友好性

## 总结

本次优化主要扩展了技能编排中意图识别模块的服务支持能力，从单一的插件支持扩展为插件+服务流的双重支持，同时完善了用户操作的提示机制，提升了平台的应用构建能力和用户体验。
